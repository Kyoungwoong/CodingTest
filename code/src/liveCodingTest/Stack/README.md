# 📘 Stack 구현하기 (Java)

## 📌 개요
이번 과제는 **스택(Stack)** 자료구조를 직접 구현하는 것이다.  
Java의 `Stack`, `Deque`, `LinkedList`, `ArrayList` 등의 라이브러리를 사용하지 않고,  
내부 구조를 직접 설계하여 LIFO 구조를 제공해야 한다.

또한 스택의 기본 기능뿐 아니라, 선택 과제로 **O(1) 시간에 최소값을 반환하는 MinStack 기능**을 구현할 수 있다.

---

## 🎯 구현 요구사항

### 구현해야 할 클래스
- `MyStack<T>` 인터페이스
- `ArrayStack<T>` 클래스 (배열 기반 스택)
- (선택) `MinStack<T extends Comparable<T>>` 클래스

---

## 🧱 기능 요구사항

### 1. 기본 스택 기능 (필수)
아래 기능을 인터페이스에 선언하고 `ArrayStack`에서 구현한다.

| 메서드 | 설명 |
|--------|------|
| `void push(T element)` | 스택 top에 요소 추가 |
| `T pop()` | top 요소 제거 후 반환 (비었으면 예외 발생) |
| `T peek()` | top 요소 반환하되 제거하지 않음 |
| `boolean isEmpty()` | 스택이 비어 있는지 여부 |
| `int size()` | 현재 저장된 요소 개수 |

---

## 📐 설계 조건

### ✔ 1. 내부 구현 방식
- **배열 기반 스택(ArrayStack)** 으로 구현한다.
- 초기 용량은 10으로 시작한다.
- 요소 추가 중 용량을 초과하면 **배열 크기를 두 배로 확장**한다.
- pop 후 배열을 줄이지는 않는다.

### ✔ 2. 예외 처리
아래 상황에서는 `IllegalStateException` 또는 `EmptyStackException`을 발생시킨다.

- 비어 있는 스택에서 `pop()` 호출
- 비어 있는 스택에서 `peek()` 호출

### ✔ 3. 제네릭(Generic) 적용
- 스택은 어떤 타입이든 저장할 수 있어야 한다.

---

## ⭐ 선택 요구사항(가산점)

### ✔ MinStack 구현
아래 기능을 추가로 제공하는 `MinStack<T extends Comparable<T>>` 클래스를 구현한다.

| 메서드 | 설명 |
|--------|------|
| `T getMin()` | 현재 스택의 최소값을 O(1) 시간에 반환 |
| push/pop 시 최소값이 갱신되도록 내부 구조 유지 |

구현 방법 예시:
- 보조 스택을 하나 더 두고 최소값만 따로 기록하는 방식
- 또는 각 노드에 "이 노드까지의 최소값"을 저장하는 방식

---

## 🧪 테스트 요구사항

### 기본 스택 테스트
- push 3회 → size 정상 증가
- pop 정상 동작 및 size 감소
- peek은 요소를 제거하지 않아야 함
- isEmpty 체크

### 예외 테스트
- 비어 있는 스택에서 pop() / peek() 호출 시 예외 발생

### 선택 테스트(MinStack)
- push 순서: 5, 3, 7, 2
    - getMin 결과는 각각 5 → 3 → 3 → 2 이어야 함
- pop 후에도 정확한 최소값 유지
